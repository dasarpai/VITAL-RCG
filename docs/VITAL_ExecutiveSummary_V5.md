The Problem VITAL Solves

Organisations do not collapse suddenly. They weaken through correlated
drifts. Three specific gaps in how organisations are currently monitored
create the conditions for this:

+-----------------------------------------------------------------------+
| **1. Each monitoring tool watches only its own function**             |
|                                                                       |
| Finance reviews cash. HR tracks attrition. Operations measures        |
| delivery. Each sees its function clearly and the rest not at all. The |
| organisation has monitoring. What it does not have is a system that   |
| watches all functions simultaneously.                                 |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **2. No tool asks the correlation question**                          |
|                                                                       |
| What does it mean when several of these are moving in the same        |
| direction at the same time? That question is where organisational     |
| crises are born. No existing management information system asks it.   |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **3. By the time the cascade is visible, it has already formed**      |
|                                                                       |
| A cash pressure causes people to leave. Attrition in critical roles   |
| degrades delivery quality. Delivery problems erode customer           |
| relationships, which deepens the cash pressure. Each function has a   |
| monitoring tool. None of those tools can see the cascade forming.     |
| VITAL can.                                                            |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **The Central Claim**                                                 |
|                                                                       |
| ***Organisations monitor functions. VITAL monitors the organisation   |
| as a system.***                                                       |
|                                                                       |
| VITAL detects organisational distress 30 to 60 days before            |
| conventional dashboards show it --- because it monitors all functions |
| simultaneously and asks the correlation question continuously.        |
+-----------------------------------------------------------------------+

What VITAL Is --- Four Precise Statements

+-----------------------------------------------------------------------+
| **A. A continuous integrated monitoring framework --- not a periodic  |
| report**                                                              |
|                                                                       |
| VITAL monitors 22 organisational functions across four tiers          |
| simultaneously, every day, reading from connected systems in real     |
| time. It does not wait for a monthly report cycle.                    |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+

|                                                                       |

| of existing systems. No new operational infrastructure. API           |
| connections poll each connected system on a defined schedule, extract |
| the required data fields, and feed them into the VITAL computation    |
| engine.                                                               |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **C. A universal framework calibrated to sector --- not a             |
| sector-specific tool**                                                |
|                                                                       |
| Every organisation has the same organisational physiology: a          |
| financial heart, a leadership brain, people-and-capability lungs, a   |
| delivery circulatory system, a compliance immune system, a technology |
| nervous system, and a revenue market lung. The architecture is        |
| universal. Thresholds and domain parameters are calibrated to sector  |
| and context.                                                          |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **D. A new class of organisational infrastructure --- continuous      |
| systemic awareness**                                                  |
|                                                                       |
| VITAL monitors not merely whether each function's metrics are         |
| acceptable, but whether the interactions between functions are        |
| producing the conditions that precede distress. The objective is not  |
| reporting. It is early diagnosis and preventive correction.           |
+-----------------------------------------------------------------------+

*An organisation must be monitored like a living system --- not through
retrospective reports, not through static KPIs reviewed annually, but
through continuous, correlated sensing of the conditions that determine
whether the organism is thriving, stable, or moving toward distress.*

The Human Sensing Architecture --- 75% AI, 25% Human

25% of VITAL's intelligence comes from structured human sensing ---
questions put to people who carry knowledge no system can capture. Five
delivery contexts ensure each question arrives at the right moment. No
new meetings are required. VITAL's sensing methodology is grounded in
the Experience Sampling Method (ESM) and Ecological Momentary Assessment
(EMA) --- an established research framework that captures responses at
contextually relevant moments, eliminating the recall bias of periodic
surveys and producing higher-fidelity signal data.

Signal Propagation --- The Research Frontier

Signal propagation modelling tracks the sequence, timing, and causal
pathway through which a signal in one function produces consequences in
another. Four primary sequences are hypothesised:

+-----------------------------------------------------------------------+
| **1. CF2 Leadership signals lead all CF consequences**                |
|                                                                       |
| Leadership signals typically precede CF3 people consequences by 3--6  |
| weeks, CF4 delivery quality decline by 4--8 weeks, and CF1 financial  |
| deterioration by 6--12 weeks. A CF2 signal is the earliest possible   |
| predictor of multi-function cascade.                                  |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **2. E1 Culture signals propagate to multiple functions               |
| simultaneously**                                                      |
|                                                                       |
| E1 psychological safety deterioration produces consequences in CF3    |
| attrition (4--8 weeks), CF5 compliance disclosure suppression (2--4   |
| weeks), and CF4 process corner-cutting (6--10 weeks) --- all at the   |
| same time. The propagation is parallel, not sequential.               |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **3. CF7 Revenue signals propagate backward and reveal leadership     |
| response style**                                                      |
|                                                                       |
| CF7 revenue decline propagates to CF2 strategic pressure, then CF3    |
| headcount decisions, then CF4 quality pressure. The pathway reveals   |
| how leadership interprets the problem.                                |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **4. Foundation Function signals propagate to Tier 1 with months of   |
| advance warning**                                                     |
|                                                                       |
| FF1 brand erosion leads CF7 pipeline contraction by 3--6 months. FF2  |
| knowledge erosion leads CF3 capability gaps at the next departure     |
| event. Monthly Foundation sensing provides the longest advance        |
| warning of any signal type in VITAL.                                  |
+-----------------------------------------------------------------------+

*These are falsifiable hypotheses. Pilot data will establish empirical
lead times specific to each organisation and sector.*
